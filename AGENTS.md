# リポジトリガイドライン

## プロジェクト概要

SlugDockは、ZennのGitHub連携リポジトリにある記事をtitleとslugで一覧化し、Markdownファイルと記事別画像フォルダへ素早くアクセスするためのローカル専用macOSアプリである。

対象利用者は開発者本人で、対象コンテンツはZennの記事とその画像に限定する。Markdown本文の編集、記事の作成・削除、Git操作、Zenn API・CLI連携、外部通信、アプリ配布は初版の対象外とする。Zennリポジトリ内のファイルを唯一の正本とし、独自DBや記事キャッシュは作成しない。永続化する業務データは、最後に選択したリポジトリルートの絶対パスなど、`SPEC.md`で定めた最小限の設定だけとする。

詳細な機能要件と振る舞いは`SPEC.md`を正本とする。
このセクションには、作業判断に必要な概要だけを記載すること。

## リポジトリ構成

リポジトリ直下の主な構成は以下のとおり。

* `SlugDock.xcodeproj`: macOSアプリと単体テストのXcodeプロジェクト。Yams 6.2.2をSwift Packageとして固定する
* `SlugDock/`: アプリターゲットのSwiftソース
* `SlugDockTests/`: XCTestによる単体テスト
* `SPEC.md`: 製品要件、振る舞い、受け入れ条件の正本
* `README.md`: アプリ概要、ビルド方法、操作方法
* `mise.toml`: Node.js 22のローカルツール設定。アプリのビルドとテストには使用しない
* `AGENTS.md`: このリポジトリで作業するエージェント向けの規則

内部の分割は`SPEC.md`の推奨コード構成を目安とし、責務が発生していないファイルやレイヤーを先回りして追加しない。

## コーディング規約

* 使用言語と環境: Swift 6、SwiftUI、AppKit、Foundation。macOS Deployment Targetは15.0、開発環境はXcode 26.3とする
* 外部依存: Front Matter解析用のYams 6.2.2だけ。XcodeプロジェクトのSwift Package設定と`Package.resolved`で固定する
* 命名規則: Swift標準の規則に従い、型名はUpperCamelCase、プロパティ・関数・ローカル変数はlowerCamelCaseとする。アプリ、プロジェクト、ターゲット、主要ディレクトリの名称は`SlugDock`に統一する
* コード構成: SwiftUIを主体とし、Finder、クリップボード、ファイル選択などmacOS固有機能だけをAppKit経由で扱う。ファイル走査、Front Matter解析、画像操作などのI/OとUI状態更新を分離するが、不要なプロトコル、汎用化、レイヤーは追加しない
* コメント方針: コードから明らかな処理の言い換えではなく、ファイル競合回避、エラー分類、MainActor境界など、判断理由や制約が必要な箇所に限定してコメントする
* Formatter: 未確定。設定ファイルと実行コマンドは存在しない
* Linter: 未確定。設定ファイルと実行コマンドは存在しない
* 型チェック: 独立した型チェックコマンドは設けず、対象ターゲットのビルドで確認する
* エラー処理: ファイル操作の失敗を黙って無視しない。Front MatterエラーとMarkdown読み取りエラーを区別し、回復可能な失敗は非モーダル表示、処理継続不能な失敗だけアラートで表示する。暗黙的なフォールバックで不正な状態を正常扱いしない

過剰な抽象化や、現在の要件に不要な拡張性を追加しないこと。

## セキュリティ

SlugDockはローカル専用で、初版では外部通信、認証情報、Zenn API、GitHub APIを使用しない。App Sandboxは無効とし、ユーザーが選択したZennリポジトリを直接読み書きする。

Finderから受け取るドロップデータは外部入力として扱い、ローカルファイルURL、通常ファイル、対応拡張子、読み取り可能性、3,000,000 bytes以下であることをコピー前に検証する。コピー先の既存ファイルは上書きせず、同名時は仕様どおり連番で別名を決定する。保存するリポジトリルートは絶対パスだけとし、記事内容や画像内容をUserDefaultsへ保存しない。

初版に認証情報や外部通信を追加しない。将来それらが必要になった場合は、`SPEC.md`の変更と明示的な承認を先に行う。秘密情報をリポジトリへ保存したり、ログやエラー表示へ出力したりしない。

## ビルドとテスト

実装と検証にはXcode 26.3を使用する。初回ビルド時はSwift Package ManagerがYams 6.2.2を取得する。

* 初期セットアップ: `SlugDock.xcodeproj`をXcode 26.3で開き、Package Resolutionの完了を待つ
* 開発実行: Xcodeで`SlugDock` Schemeと`My Mac`を選択し、`⌘R`で実行する
* ビルド: `xcodebuild build -project SlugDock.xcodeproj -scheme SlugDock -destination 'platform=macOS'`
* Formatter: 未確定。未導入
* Lint: 未確定。未導入
* 型チェック: 独立コマンドは設けず、上記ビルドで確認する
* 単体テスト: `xcodebuild test -project SlugDock.xcodeproj -scheme SlugDock -destination 'platform=macOS'`
* 統合テスト: 自動化設定は存在せず、`SPEC.md`の手動確認項目に従う

存在しないコマンドを推測で記載しないこと。

変更後は、変更範囲に対応する検証を実行すること。
検証を実行できなかった場合は、その理由と未確認範囲を報告すること。

## 外部ライブラリとドキュメント

外部ライブラリ、フレームワーク、SDK、APIの仕様が実装に影響する場合は、対象バージョンに対応する公式ドキュメントまたは公式リポジトリを優先して確認すること。

Yamsは公式リポジトリの6.2.2向けドキュメントを参照する。AppleのフレームワークとAPIは、macOS 15.0のDeployment TargetおよびXcode 26.3に対応するApple公式ドキュメントを参照する。

依存関係はYamsだけに限定し、XcodeプロジェクトのSwift Package設定と`Package.resolved`で使用バージョンを固定・確認する。新しい外部依存が必要になった場合は、追加前に`SPEC.md`との整合性とFoundation、SwiftUI、AppKitで代替できない理由を確認する。

## 要件と変更範囲

* 製品要件と振る舞いは`SPEC.md`を正本とする
* 指示されていない機能を追加しない
* タスクに無関係なリファクタリングを行わない
* 既存の公開インターフェースを変更する場合は影響範囲を確認する
* 仕様と既存実装が矛盾する場合は、黙って既存実装へ合わせず矛盾を報告する

## 完了条件

変更は以下を満たした時点で完了とする。

* 指定された要件を満たしている
* 関係するビルド、Lint、型チェック、テストを通過している
* 不要な変更が含まれていない
* エラーや未確認事項が明示されている
* 必要なドキュメントが更新されている
